## Popular Endpoints

Explore our most commonly used API endpoints to get started quickly.

### Find Plant by ID

<div>
**GET** [`/plant/{plantId}`](/api-reference/plant/getPlantById)
</div>

Retrieve detailed information about a specific plant using its unique identifier.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `plantId` | integer | Yes | The unique identifier of the plant |

<br/>

<Tabs>
  <Tab title="Request">
```http
GET /plant/101
```
  </Tab>
  <Tab title="Response">
```json
{
  "id": 101,
  "name": "Fern",
  "status": "available",
  "tags": ["green", "leafy"]
}
```
  </Tab>
</Tabs>

---

### Add New Plant

<div>
**POST** [`/plant`](/api-reference/plant/addPlant)
</div>

Add a new plant to your inventory with complete details.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | The name of the plant |
| `status` | string | Yes | Availability status (`available`, `pending`, `sold`) |
| `tags` | array | No | Tags for categorization |

<br/>

<Tabs>
  <Tab title="Request">
```json
{
  "name": "Monstera Deliciosa",
  "status": "available",
  "tags": ["tropical", "indoor", "popular"]
}
```
  </Tab>
  <Tab title="Response">
```json
{
  "id": 102,
  "name": "Monstera Deliciosa",
  "status": "available",
  "tags": ["tropical", "indoor", "popular"]
}
```
  </Tab>
</Tabs>

---

### Search Plants by Status

<div>
**GET** [`/plant/findByStatus`](/api-reference/plant/findPlantsByStatus)
</div>

Find all plants matching a specific availability status.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `status` | string | Yes | Filter by status (`available`, `pending`, `sold`) |

<br/>

<Tabs>
  <Tab title="Request">
```http
GET /plant/findByStatus?status=available
```
  </Tab>
  <Tab title="Response">
```json
[
  {
    "id": 101,
    "name": "Fern",
    "status": "available"
  },
  {
    "id": 102,
    "name": "Monstera",
    "status": "available"
  }
]
```
  </Tab>
</Tabs>

## Authentication

The Plant Store API supports multiple authentication methods to suit your integration needs.

<Accordion title="API Key Authentication">
Include your API key in the request header for simple integrations.

```bash
curl -H "X-API-Key: your_api_key_here" \
  https://api.plantstore.com/v1/plants
```
</Accordion>

<Accordion title="OAuth 2.0">
For production applications, we recommend OAuth 2.0 with the following scopes:

| Scope | Description |
|-------|-------------|
| `read:plants` | Read plant catalog |
| `write:plants` | Create and update plants |
| `read:orders` | View order information |
| `write:orders` | Create and manage orders |
</Accordion>
